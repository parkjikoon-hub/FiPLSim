## 2. 실제 배관 구조 (확정)

### 단방향 (uni) — 교정 후
```
교차배관(80A) ─→ [80/65A 이경엘보 90°] ─→ [65/50A 레듀서니플] ─→ 50A 가지배관 →
```
- T분기이음쇠 **없음** (K3 제거)
- 65A 입구관(0.3m) **없음** (엘보→레듀서 직결)
- 이경엘보가 **방향전환(90°) + 구경축소(80→65)** 동시 수행

### 양방향 (bi) — 변경 없음
```
← 50A가지배관 ─[레듀서]─ 65A입구관(≤0.3m) ─[T분기이음쇠]─ 65A입구관(≤0.3m) ─[레듀서]─ 50A가지배관 →
                                                  │
                                        교차배관(80A) ═══→
```

## 4. 핵심 기술 정보

### K값 근거 원칙 (사용자 확인)
- **NFPA 13 기준 필수** — Crane TP-410은 레듀서에만 사용
- K_TEE_BRANCH_basis.md와 동일 방법론: `K = f_T × (L_eq/D)`
- NFPA 13은 이경엘보와 동경엘보를 구분하지 않음

### 코드 구조 핵심
- `pipe_network.py` 함수들은 이미 파라미터화:
  - `_calculate_branch_profile(K3_val=...)`: 전달받은 K값 사용
  - `calculate_dynamic_system(K3_val=...)`: 전달받은 K값 사용
  - `generate_dynamic_system(branch_inlet_config=...)`: 전달받은 config 사용
- `inlet_pipe_length_m=0.0`이면 → 마찰손실 0, K값은 정상 적용, 레듀서도 정상 적용
- `run_uni_bi_final.py`의 `run_det()` 내부에서 uni/bi 토폴로지 분기 이미 존재

### run_uni_bi_final.py 토폴로지 처리 (현재)
- **uni** (Line 222~246): num_br=4, 전체 유량, K3 전달
- **bi** (Line 247~275): num_br=2, 유량 절반, equip_loss + tee_loss 선차감 후 K3 전달
- 수정 후: uni에서 K3 대신 K_ELBOW_90_65A 전달, config도 변경

### 예상 영향
- uni 분기 입구 K값: 1.0 → 0.53 (**-47%**)
- uni 65A 입구관 마찰: 제거 (추가 손실 감소)
- P_REF 하락 예상 (uni 손실 감소)
- uni-bi 격차 확대 가능 (uni가 더 유리해짐)

## 6. 주의사항

1. **K값은 반드시 NFPA 13 기준**: Crane TP-410은 레듀서에만 사용. 이경엘보 K값도 NFPA 13 등가길이로 산출해야 일관성 유지.
2. **양방향은 변경 없음**: 현재 코드의 양방향 모델(T분기 + 65A 입구관)은 정확함.
3. **pipe_network.py 수정 금지**: 이미 파라미터화되어 있으므로, 호출부(run_uni_bi_final.py)에서 다른 값을 전달하면 됨.
4. **기존 시뮬레이션 결과는 수정 전 데이터**: 재시뮬레이션 후 덮어써짐. 수정 전후 비교가 필요하면 미리 백업.
5. **물리상수는 run_uni_bi_final.py 내부 하드코딩**: constants.py 기본값과 무관하게 논문값(ε=0.046, ρ=1000, ν=1.002e-6) 사용.
